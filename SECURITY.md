# Security Policy

## Supported versions

Clipora is currently a pre-1.0 public preview. Security fixes are provided for the latest published release only.

## Reporting a vulnerability

Please do not disclose vulnerabilities in a public issue, discussion, or pull request.

Use GitHub's private vulnerability reporting from the repository's **Security** tab and include:

- affected Clipora version;
- Windows version;
- impact and realistic attack scenario;
- minimal reproduction steps;
- any suggested mitigation.

Do not attach real clipboard databases, private clipboard contents, credentials, or personal files. Use synthetic test data and redact local usernames and paths.

You should receive an initial acknowledgement within seven days. A fix and disclosure timeline will depend on severity and reproducibility.
