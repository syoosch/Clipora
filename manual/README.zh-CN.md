## 安装与首次运行

1. 打开[最新发布页面](https://github.com/syoosch/Clipora/releases/latest)，下载 `Clipora-0.4.3-setup.exe`。
2. 运行安装包。Clipora 仅为当前 Windows 用户安装，不需要管理员权限。
3. 当前安装包未做代码签名。若 SmartScreen 出现提示，请选择“更多信息”，将安装包名称和 SHA-256 与 Release 中的信息对照，仅在确认文件可信后继续运行。
4. 让 Clipora 在托盘中保持运行，它才能持续监听新的剪贴板变化。

Clipora 默认将数据存放在 `%LOCALAPPDATA%\Clipora`，之后可以在设置中选择自定义数据位置。

## 主窗口与窗口行为

- 按 `Alt+V` 显示主面板。
- 标题栏提供搜索/筛选、窗口置顶、最小化、设置和关闭入口。
- 历史记录按时间倒序排列，每 8 小时为一个可折叠时间段；置顶卡片固定显示在顶部区域。
- 卡片显示识别类型、来源应用、记录时间和内容预览，悬停后显示操作按钮。
- 最小化时按当前设置执行；默认会变为可在屏幕边缘移动的贴边悬浮条，鼠标靠近或点击即可恢复。
- 关闭窗口通常只会把 Clipora 收入通知区域。需要重新打开或彻底退出时使用托盘菜单。
- 设置中可以调整是否显示任务栏图标、窗口置顶、静默启动和开机启动。

## 捕获剪贴板内容

Clipora 被动监听正常的剪贴板变化，不会替代 `Ctrl+C` 或 `Ctrl+V`。当前支持：

- 普通文字和富文本；
- HTTP/HTTPS URL；
- 代码和颜色值；
- 图片和截图；
- 文件和文件夹。

也可以把文件、文字、图片或富文本直接拖入面板。手动拖入属于用户明确导入，即使暂停了被动记录，仍然可以保存。

单条大小上限默认是 25MB，可以在设置中修改。超过上限的文字或图片不会保存；低于上限的文件会在条件允许时复制到 Clipora 数据目录，文件夹和较大的文件集合则只保存原始路径引用。

开启重复合并后，再次从外部复制或拖入相同内容会把已有记录移到前面，而不是创建重复卡片。从 Clipora 内部重用内容不会改变原卡片的记录时间。

## 查找与整理历史

- 点击标题栏的搜索按钮展开关键词搜索与筛选区。
- 筛选区可以在类型筛选和标签筛选之间切换。
- 启用 OCR 后，Windows 本地 OCR 识别出的图片文字也能参与搜索。
- 可以从筛选区或设置中新建标签，并在标签管理中改名、调色、排序或删除。
- 置顶项目固定显示在顶部，也不会被保存周期自动清理。
- 删除会先把项目移入回收站；可以恢复，也可以在设置中永久清空。
- 浏览很长的历史列表时，可选的“回到顶部”按钮会在向上滚动后出现。

## 重用、打开与拖出内容

### 重用卡片

点击卡片会把内容写回剪贴板。启用自动粘贴后，Clipora 还会返回之前的窗口并发送粘贴；若自动粘贴未能完成，内容仍保留在剪贴板中，可以手动粘贴。

### 卡片操作

- 文字、富文本、代码和颜色卡片会打开可选取文字的查看层。
- 图片和普通文件可以交给 Windows 默认应用打开。
- URL 卡片只允许打开绝对 HTTP/HTTPS 链接。
- 可执行文件、脚本、快捷方式、注册表文件等主动类型在打开前会显示确认对话框，默认选择是取消。

### 拖出

可以把支持的卡片拖到其他应用或文件资源管理器。仅引用文件会在重用或开始拖出时检查原路径；若原文件被移动或删除，请恢复到原位置或重新捕获。

## 全局快捷键

| 操作 | 默认值 | 行为 |
| --- | --- | --- |
| 打开面板 | `Alt+V` | 显示或激活 Clipora。 |
| 纯文本粘贴 | `Ctrl+Shift+V` | 去除格式后粘贴到之前的应用。 |
| 顺序粘贴 | 未设置 | 把一组捕获内容按顺序逐条粘出。 |

三个快捷键都可以在设置中修改。若新组合已被 Windows 或其他应用占用，Clipora 会显示冲突并保留之前有效的快捷键。Clipora 不会接管系统原有的 `Ctrl+C`、`Ctrl+V` 或 `Win+V`。

顺序粘贴到达最后一项后会停止，不会从头循环。捕获或导入新内容后再开始下一轮。

## 设置

### 常规

- 点击卡片后的自动粘贴；
- 图片 OCR；
- 开机启动和静默启动；
- 窗口置顶、任务栏显示、最小化与关闭行为；
- “回到顶部”按钮。

### 存储

- 保存周期：1、3、7、30 天或永久，默认 3 天；
- 单条大小上限，默认 25MB；
- 重复合并；
- 数据目录迁移；
- 永久清空回收站。

### 快捷键

录制、重置并检查打开面板、纯文本粘贴和顺序粘贴三个快捷键的冲突。

### 隐私

暂停被动记录，或把指定的运行中应用加入排除名单。

### 外观

选择跟随系统、浅色或深色；在 Fluent 与液态玻璃之间切换；为 Fluent 选择本地背景并调整透明度，或调整液态玻璃通透度。

### 备份与恢复

导出 `.clpbak` 文件，或检查并把可信备份合并到当前历史。

## 文件与存储

Clipora 对文件采用智能存储：

- 较小的文件集合会复制进受管数据目录，因此即使原文件后来被删除，卡片仍可继续使用。
- 文件夹和较大的文件集合只保存路径引用，占用的受管空间更少，但依赖原路径。
- 仅引用项目不会在每次加载列表时扫描原路径，只会在重用或拖出时检查。

安装版默认使用 `%LOCALAPPDATA%\Clipora`。选择新数据位置后，Clipora 会先复制并完整校验，再切换到新目录。迁移失败时继续使用旧目录，不会自动合并或删除任一位置。

若已配置的数据目录在启动时不可用，Clipora 会要求重试、使用默认位置或退出，不会静默切换数据目录。

## 隐私控制

- Clipora 没有剪贴板上传或网络服务，历史记录保存在本机。
- 暂停会停止被动剪贴板记录，但用户明确拖入仍然可用。
- 应用排除名单会阻止指定进程的内容被动进入历史。
- Clipora 尊重 Windows 的“排除剪贴板历史”标记；若无法可靠确认该标记，内容会被跳过。
- SQLite 数据库、附件和 `.clpbak` 备份均未加密，同一 Windows 用户下运行的其他进程可能读取这些文件。

数据类型、保留策略、诊断、删除和备份保护方式详见[隐私说明](../PRIVACY.md)。

## 备份与恢复

### 导出

进入设置，选择“备份与恢复”，导出 `.clpbak` 文件。备份包含剪贴板原始内容且未加密，只应保存在可信位置。

### 恢复

1. 选择自己创建或来源可信的备份。
2. 查看备份创建时间和条目数。
3. 确认合并导入。

导入会在改变当前数据前校验归档。合法项目会被合并，重复项目会跳过，现有历史不会被删除。备份被拒绝通常表示文件已损坏、不兼容或结构不安全，不要通过修改归档来绕过校验。

## 卸载与数据保留

可以从开始菜单或 Windows 设置卸载 Clipora。若托盘实例仍在运行，卸载程序会在确认后关闭它。

默认卸载会保留剪贴板数据，方便以后重新安装继续使用。只有确定要永久清除时才选择删除全部 Clipora 数据；删除前先导出仍需保留的备份。

## 常见问题

### 新内容没有被记录

- 确认 Clipora 仍在托盘中运行。
- 检查是否暂停记录，或来源应用是否在排除名单中。
- 来源可能设置了 Windows 的“排除剪贴板历史”标记。
- 内容可能超过当前大小上限。
- 无法安全确认隐私状态时，Clipora 会跳过该内容。

### 文件卡片无法重用或拖出

该卡片可能只保存了路径引用，而原文件已经移动或删除。请恢复原路径，或重新捕获文件。

### 快捷键没有反应

进入快捷键设置查看冲突提示。如果组合已被 Windows 或其他应用占用，请改用其他组合。

### 主窗口不见了

先查看通知区域。若 Clipora 最小化到了屏幕边缘，把鼠标移到贴边悬浮条或点击它即可恢复面板。

### SmartScreen 阻止安装

只从官方 Release 页面下载安装包，将 SHA-256 与 Release 中的校验值对照，并仅在确认文件可信时继续。当前安装包未做代码签名。

### 已配置的数据目录丢失

重新连接磁盘或恢复目录后重试。只有确实希望用独立的默认数据根重新开始时才选择默认位置；丢失的原目录不会被删除。

### 备份被拒绝

请使用原始备份文件，并确认它来自可信的 Clipora 安装。损坏、不兼容或结构不安全的归档会在导入前被主动拒绝。

发现问题时请使用仓库提供的结构化 [Issue 表单](https://github.com/syoosch/Clipora/issues)，不要附带私人剪贴板内容或个人文件。安全漏洞请按[安全策略](../SECURITY.md)私下报告。
